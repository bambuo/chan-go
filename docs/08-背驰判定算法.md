# 背驰判定算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 输入

- `价格极值[]`：价格高点序列（或低点序列）。
- `MACD柱[]`：MACD柱状图（红柱/绿柱）。

> 背驰判定的两种方法：
> 1. **MACD柱面积法**（第24课）—— 最稳定，推荐用于程序化实现。
> 2. **均线吻面积法**（第15课）—— 前后两个同向"吻"之间短期/长期均线相交围成的面积，后面积 < 前面积即为背驰。改进：**缠中说禅趋势平均力度** = 趋势力度 ÷ 吻的持续时间，可提前判断。

### 算法步骤

背驰判定基于 **缠论第53课** 的标准定义：**比较最后一个中枢的进入段与离开段的累积强度**。

对每个走势类型，取其最后一个中枢，构建进入段和离开段：

- **进入段**：走势类型起点到最后一个中枢起点之间（不含中枢第一笔）的笔序列。
- **离开段**：最后一个中枢终点之后到走势类型终点（不含中枢最后一笔）的笔序列。

**顶背驰检查**

若走势方向向上（上涨趋势），比较进入段与离开段的累积 MACD 面积：

- 离开段最高价 > 进入段最高价（价格创新高）。
- 计算进入段的 MACD 柱面积（取绝对值之和）。
- 计算离开段的 MACD 柱面积（取绝对值之和）。
- 若离开段面积 < 进入段面积 × 0.95（面积缩小 5% 以上），则判定为顶背驰。

记录的信息包括：背驰位置、进入段最高价、离开段最高价、MACD 面积比。

**底背驰检查**

若走势方向向下（下跌趋势），比较进入段与离开段的累积 MACD 面积：

- 离开段最低价 < 进入段最低价（价格创新低）。
- 若离开段面积 < 进入段面积 × 0.95，判定为底背驰。

> **重要**：背驰必须发生在趋势中（至少两个同向中枢），盘整中不存在趋势背驰。盘整背驰的判定见下文。

### 盘整背驰判定

盘整走势（仅含一个中枢）中的背驰是指离开中枢的笔与进入中枢的笔之间的 MACD 面积比较：

- **进入笔**：走势进入中枢的最后一笔（方向指向中枢）。
- **离开笔**：走势离开中枢的第一笔（方向离开中枢）。
- 若离开笔的 MACD 面积 < 进入笔的 MACD 面积 × 0.95，则判定为盘整背驰。

> 盘整背驰与趋势背驰的区别：盘整背驰不保证走势一定转折（可能只是中枢震荡延续），而趋势背驰（至少两个同向中枢后的背驰）是走势结束的充分条件。

### 区间套算法（精确定位）

从高级别背驰点开始，逐级向下找精确转折点：

已在高级别确认一个背驰点 → 获取该背驰点所处的时间范围 → 在次级别数据的时间范围内检测背驰 → 取最接近范围末尾的背驰点 → 将范围缩小到该点 → 继续向下一个级别重复。最后一个级别的背驰点即为精确转折点。
