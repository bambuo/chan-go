# K线包含处理算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 输入

- `原始K线[]`：原始K线序列，每根K线包含 `(高, 低, 开盘, 收盘)`。

### 输出

- `合并K线[]`：包含处理后的合并K线序列 `(高, 低)`。

### 算法步骤

从第一根K线开始，将其加入结果序列，记录当前方向为向上。

依次处理后续每根原始K线：

**判断包含关系**

若当前K线与结果序列中最后一根K线之间存在包含关系——
即 `(当前高 ≤ 前一根高 且 当前低 ≥ 前一根低)` 或 `(当前高 ≥ 前一根高 且 当前低 ≤ 前一根低)`，
则判定为包含，进行合并：

- 若当前方向为向上，合并后取两边高点中的更高者、低点中的更高者。
- 若当前方向为向下，合并后取两边高点中的更低者、低点中的更低者。

合并后的K线替换结果序列中的最后一根，继续与下一根比较。

若不存在包含关系，则当前K线直接加入结果序列末尾。

**更新方向**

以非包含序列中最后两个元素为基准判定方向（缠论原文 AND 条件）：
- 若 `第二根高点 > 第一根高点 且 第二根低点 > 第一根低点` → 方向更新为向上。
- 若 `第二根高点 < 第一根高点 且 第二根低点 < 第一根低点` → 方向更新为向下。
- 否则方向保持不变（按向上处理）。

处理完所有K线后，结果序列即为包含处理后的合并K线序列。

### 确定性说明

- 包含关系判定：`(当前高 ≤ 前一根高 且 当前低 ≥ 前一根低) 或 (当前高 ≥ 前一根高 且 当前低 ≤ 前一根低)`
- 方向判定：缠论原文 AND 条件（High 和 Low 同时同向）
- 合并后的K线继续与下一根检查包含，迭代进行
