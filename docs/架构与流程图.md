### 全景图读图要点

1. **唯一的输入边界**是 Redis Stream；**唯一的真值**是 stream 里的 K 线。引擎不认识交易所。
2. **主链路**（蓝色→绿色→蓝色）：`M1 输入 → M2 计算 → M3/M4/M5/M6 结构与信号 → M8 输出`。
3. **横向能力**（橙色）贯穿所有层：M7（状态）被所有模块读写；M0（快照）由 M7 派生；M9（回测）通过向 Redis 灌历史 K 线切入；M10（可观测）监听全部。
4. **回测与生产同代码路径**：回测只是把"实盘数据适配层"换成"历史回放"，引擎内部完全无感知。

## 二、交互流程图（按场景）

### 场景 1：实时信号产出（最常见，主路径）

> 一根新 K 线从到达到机器人收到信号的完整链路。**这是引擎 99% 时间在做的事。**

```mermaid
sequenceDiagram
    autonumber
    participant ADAPT as 数据适配层<br/>(外部)
    participant STREAM as Redis Stream
    participant G1 as M1 输入网关
    participant M2 as M2 缠论核心
    participant M3 as M3 结构树
    participant M4 as M4 递归级别
    participant M5 as M5 信号引擎
    participant M6 as M6 共振引擎
    participant M8 as M8 输出网关
    participant ROBOT as 🤖 机器人
    participant M0 as M0 快照层

    ADAPT->>STREAM: XADD 写入 K 线<br/>(ts 单调递增)
    STREAM->>G1: XREADGROUP 消费
    G1->>G1: 校验 ts（R17）<br/>倒退/重复→拒绝记 audit<br/>缺口→标记<br/>字段异常→拒绝
    G1->>M2: 推送合法 K 线

    Note over M2,M3: 增量计算（R16）<br/>90% 场景：只延伸尾部<br/>10% 场景：触发回溯修正

    M2->>M2: 增量处理<br/>(包含→分型→笔→线段→中枢→走势→背驰)
    alt 场景 A：尾部延伸
        M2-->>M3: 更新最后元素
    else 场景 B：回溯修正
        M2->>M3: 撤销受影响窗口 + 重算<br/>M3 记录 diff + lineage 映射
    end

    M3->>M4: 结构变更事件
    M4->>M4: 双轨更新（实时轨/确认轨）<br/>检测级别漂移

    M4->>M5: 结构变更
    M5->>M5: 识别买卖点<br/>计算 anchor（lineage）<br/>去重：anchor 已存在→更新，不存在→新建
    M5->>M5: 计算 confidence(§8.4) + strength(§8.5)<br/>更新 targets(§8.7)<br/>状态机流转

    M5->>M6: 信号事件
    M6->>M6: G-2 区间套 / G-1 跨层判定<br/>共振等待窗口管理（R2）

    M6->>M8: 信号 + 共振事件
    M8->>ROBOT: WS Push<br/>(signal.created / state_changed /<br/> invalidated / resonance.triggered)

    Note over M0: 串行化快照（R29/B1）<br/>K 线处理完后检查 pending_snapshot
    M0->>M0: 若 pending=true 则快照<br/>(含 Redis offset)
```

**关键点**：
- 步骤 4-5：M1 的校验是引擎的"门神"，垃圾 K 线进不来（R17）
- 步骤 7-9：M2 的增量计算 + M3 的回溯修正是性能与正确性的核心（R16）
- 步骤 14-15：M5 的 anchor 去重保证同一信号不被重复 created（R13）
- 步骤 21：WS 推送是异步的，REST 也可随时查询最新快照

### 场景 2：崩溃恢复（R19）

> 引擎进程意外挂掉后的完整恢复链路。

```mermaid
flowchart TB
    CRASH["💥 进程崩溃<br/>进程内状态（M7）全丢"]

    CRASH --> START["进程重启"]

    START --> READ_SNAP["M0 读取 snapshots/{symbol}/latest.json"]

    READ_SNAP --> VALIDATE{"快照完整性校验<br/>(R29/B4)<br/>① schema 完整<br/>② offset 非负整数<br/>③ offset ≤ XLEN<br/>④ offset 对应 ts ≥ 快照最后 K 线 ts"}

    VALIDATE -->|通过| RESTORE["M0 → M7 恢复进程状态<br/>结构树当前版本<br/>未结束信号<br/>双轨状态<br/>Redis 消费 offset"]

    VALIDATE -->|不通过| FALLBACK1["回退到上一个快照<br/>snapshots/{symbol}/{prev_ts}.json"]
    FALLBACK1 --> VALIDATE

    VALIDATE -->|"全部损坏<br/>(Q3.2 = c)"| REPLAY["兜底：从 Redis Stream<br/>最早可用 offset 全量重灌<br/>(依赖 MAXLEN ≥ 80000)"]
    REPLAY --> RESTORE

    RESTORE --> CONSUME["M1 从快照 offset 续消费<br/>XGROUP SETID 重置 cursor<br/>(R29/D2)"]

    CONSUME --> CATCHUP["增量处理后续 K 线<br/>通常几秒追上实时流"]

    CATCHUP --> READY["✅ 恢复完成，进入正常工作<br/>（无 warming_up 状态，R12）"]

    %% 样式（高对比）
    classDef crash fill:#ef5350,stroke:#b71c1c,stroke-width:2px,color:#ffffff
    classDef check fill:#fff176,stroke:#f57f17,stroke-width:2px,color:#4e342e
    classDef ok fill:#9ccc65,stroke:#33691e,stroke-width:2px,color:#1b5e20
    classDef fallback fill:#ffb74d,stroke:#e65100,stroke-width:2px,color:#3e2723

    class CRASH crash
    class VALIDATE check
    class RESTORE,CONSUME,CATCHUP,READY ok
    class FALLBACK1,REPLAY fallback
```

**恢复策略三档**（优先级从高到低）：
1. **快照优先**（正常情况，秒级恢复）
2. **回退上一快照**（最新快照损坏，损失几分钟状态）
3. **全量重灌**（所有快照都坏，最慢但最稳，依赖 Redis MAXLEN）

### 场景 3：级别漂移与信号 recast（R20 / §10.3）

> 真递归级别下，"L2 走势被重分类为 L3 的一部分"时，引擎如何处理已发出的信号。

```mermaid
sequenceDiagram
    autonumber
    participant M4 as M4 递归级别
    participant M5 as M5 信号引擎
    participant M6 as M6 共振引擎
    participant M8 as M8 输出网关
    participant ROBOT as 🤖 机器人

    Note over M4: 实时轨 vs 确认轨分歧<br/>例如：原以为 L2 走势结束<br/>实际是 L3 走势的一部分

    M4->>M4: 检测到级别漂移<br/>level_recast 事件
    M4->>M5: 通知受影响信号

    loop 每个受影响的旧信号
        alt 旧信号状态 = candidate（未确认）
            M5->>M5: 标记 invalidated<br/>(撤回)
            M5->>M5: 基于新级别创建新 candidate<br/>(recast_from 指向旧信号)
            M5->>M6: 新信号
            M6->>M8: signal.recast 事件
            M8->>ROBOT: 推送（旧 invalidated + 新 candidate）
        else 旧信号状态 = confirmed（已确认）
            M5->>M5: 标记 superseded（不撤回）
            M5->>M5: 发新级别信号
            M5->>M6: recast 事件 + 新信号
            M6->>M8: signal.recast + 新信号
            M8->>ROBOT: 推送（机器人自行决定<br/>如何处理已下仓位）
        end
    end
```

**关键设计**：
- candidate 撤回（没下仓位，干净重发）
- confirmed 不撤回（已下仓位，机器人自己决定）—— **符合"引擎报告，机器人决策"哲学**

### 场景 4：双轨分歧降级（R20 / §14.5.4）

> 极端行情下，实时轨与确认轨长期不一致时的降级处理。

```mermaid
flowchart TB
    DETECT["M4 检测：实时轨 vs 确认轨<br/>分歧持续 > 阈值<br/>(默认 = 该级别平均走势时长 × 2)"]

    DETECT --> ISOLATE["故障隔离（R20/Q4.2）<br/>仅影响当前 symbol<br/>其他 symbol 照常工作"]

    ISOLATE --> DEGRADE["进入降级模式（R20/Q4.1）"]

    DEGRADE --> D1["① 确认轨暂停推送<br/>避免给机器人矛盾信息"]
    DEGRADE --> D2["② 只推实时轨"]
    DEGRADE --> D3["③ 受影响信号 risk 拉满<br/>provisional = true<br/>recast_risk ≥ 0.5<br/>confidence / strength 自然下降"]
    DEGRADE --> D4["④ M10 告警<br/>提示运维关注"]

    D1 & D2 & D3 & D4 --> WAIT["等待双轨收敛"]

    WAIT --> CONVERGE{"双轨重新收敛？"}
    CONVERGE -->|是| RECOVER["恢复正常推送<br/>受影响信号走 recast 流程（场景3）"]
    CONVERGE -->|否| WAIT

    %% 样式（高对比）
    classDef detect fill:#ef5350,stroke:#b71c1c,stroke-width:2px,color:#ffffff
    classDef degrade fill:#ffb74d,stroke:#e65100,stroke-width:2px,color:#3e2723
    classDef recover fill:#9ccc65,stroke:#33691e,stroke-width:2px,color:#1b5e20

    class DETECT,CONVERGE detect
    class ISOLATE,DEGRADE,D1,D2,D3,D4,WAIT degrade
    class RECOVER recover
```

**机器人可见性**：故障期间机器人通过 `provisional=true` + 高 `recast_risk` 自然感知风险，不需要特殊协议。

## 三、模块职责速查表（配合全景图阅读）

| 模块 | 层 | 职责一句话 | 关键约束 |
|---|---|---|---|
| **数据适配层** | 外部 | 接交易所，产出干净 K 线写 Redis | 不在引擎范围 |
| **M1 输入网关** | L1 | 消费 Redis，校验 K 线，路由到 symbol | 唯一入口；R17 严格校验 |
| **M2 缠论核心** | L2 | 14 份算法的纯函数实现 | 纯函数 + 增量 + 回溯修正（R16）|
| **M3 结构树** | L2 | 不可变版本化结构存储 | lineage 跨版本追踪（§10.5）|
| **M4 递归级别** | L2 | 双轨制多级别构建 | 级别漂移检测 → recast |
| **M5 信号引擎** | L2 | 买卖点识别 + 状态机 + confidence/strength | anchor 去重（R13）|
| **M6 共振引擎** | L2 | G-2 区间套 + G-1 跨层 + A3 过滤 | G-2 最高优先 |
| **M7 状态存储** | 横向 | 进程内状态 | 被所有模块读写 |
| **M0 快照层** | 横向 | JSON 持久化 + 恢复 | 含 Redis offset；串行化（R29/B1）|
| **M8 输出网关** | L3 | REST + WS 对外 | 唯一出口；R6 限流 |
| **M9 回测引擎** | 横向 | 灌历史 K 线 + 统计 | 同代码路径（§13.3）|
| **M10 可观测** | 横向 | metrics + audit + 告警 | 监听全部模块 |
