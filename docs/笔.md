# 笔识别算法规范

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

---

## 1. 基本对象

### 1.1 合并K线

笔识别基于包含处理后的合并K线，记为 `合并K线`。

每个 `合并K线` 至少包含：

| 字段 | 含义 |
| --- | --- |
| `索引` | 合并K线索引，从 0 递增 |
| `高` | 合并K线最高价 |
| `低` | 合并K线最低价 |
| `分型` | 分型类型：`顶`、`底`、`未知` |
| `方向` | 合并K线方向：`上`、`下` 等 |
| `前一根` | 前一根合并K线 |
| `下一根` | 后一根合并K线 |
| `原始列表` | 该合并K线包含的原始K线列表 |

辅助计算：

- 合并K线最高价：取其 `原始列表` 中各原始K线的最高价的最大值。
- 合并K线最低价：取其 `原始列表` 中各原始K线的最低价的最小值。
- 缺口判定：若两根相邻合并K线的真实价格区间（各自最高价到最低价）完全不重叠，则判定为存在缺口。

### 1.2 分型类型

| 类型 | 含义 |
| --- | --- |
| `顶` | 顶分型 |
| `底` | 底分型 |
| `未知` | 非分型 |

### 1.3 笔对象

每条笔至少包含：

| 字段 | 含义 |
| --- | --- |
| `起点合并K线` | 笔起点合并K线 |
| `终点合并K线` | 笔终点合并K线 |
| `索引` | 笔序号 |
| `方向` | 笔方向：`上` 或 `下` |
| `是否确认` | 是否为确认笔 |
| `确认终点[]` | 虚笔曾经替换过的确认终点 |
| `前一笔` | 前一笔 |
| `后一笔` | 后一笔 |

方向规则：

笔的方向由起点分型决定：起点为底分型时笔方向为向上，起点为顶分型时笔方向为向下。

价格规则：

- 向上笔的起点价格取起点合并K线的低点，终点价格取终点合并K线的高点。
- 向下笔的起点价格取起点合并K线的高点，终点价格取终点合并K线的低点。

合法性检查：向上笔必须确保起点低点 < 终点高点；向下笔必须确保起点高点 > 终点低点。

## 2. 配置项

### 2.1 配置表

| 配置 | 默认值 | 含义 |
| --- | --- | --- |
| `笔算法` | `正常` | 笔算法：`正常` 或 `顶底即成笔` |
| `是否严格` | `真` | 是否使用严格跨度规则 |
| `笔分型检查` | `半` | 分型有效性检查方式 |
| `缺口计为K线` | `真` | 缺口是否按额外 K 线计入跨度 |
| `笔终点为峰值` | `真` | 笔终点是否必须为区间极值 |
| `允许次峰值` | `真` | 是否允许次高/次低修正逻辑被跳过 |

### 2.2 笔算法

| 取值 | 含义 |
| --- | --- |
| `正常` | 正常笔，需要检查跨度、分型有效性、终点峰值 |
| `顶底即成笔` | 顶底分型即成笔，不检查跨度，但仍检查分型有效性和终点峰值 |

### 2.3 笔分型检查

| 取值 | 含义 |
| --- | --- |
| `宽松` | 只检查两个分型中间 K 线自身 |
| `半` | 检查半分型，默认值 |
| `严格` | 检查两个分型的三元素 |
| `完全` | 要求两个分型区间完全分离 |

---

## 3. 主状态机

笔识别是一个逐步推进的状态机，维护以下核心状态：

- `笔列表[]`：已生成的笔（可能包含最后一条虚笔）。
- `最后终点`：最后一笔的尾部分型（合并K线）。
- `候选分型列表[]`：第一笔生成前暂存的候选分型。

### 3.1 总入口

输入：一根已可确认分型的合并K线（倒数第二根），以及最新合并K线（用于生成或更新虚笔）。可选择是否计算虚笔。

逻辑：先执行确认笔更新流程。若需要计算虚笔，再执行虚笔更新流程。返回是否有任何变化。

### 3.2 确认笔更新流程

每次计算确认笔前，先清理上一轮产生的虚笔。

对输入的合并K线：

- **若无分型**：检查最后一笔的终点K线索引是否有变化即可。
- **若笔列表为空或最后终点为空**：尝试用该合并K线创建第一笔。
- **若该合并K线与最后终点类型相同**（同为顶或同为底）：尝试更新最后一笔的终点。
- **若该合并K线可与最后终点成笔**：创建一条新确认笔，将该合并K线设为新的最后终点。
- **若可触发更新峰值修正**：执行修正逻辑。

---

## 4. 第一笔生成

第一笔不是直接从最早两个不同分型生成，而是先缓存若干个候选分型，以便找到更合适的起点。

当笔列表为空时，每个到来的合并K线如果是有分型的，先进入 `候选分型列表`。每当新分型到来，检查它是否与 `候选分型列表` 中的异类分型成笔。若能成笔，则创建第一笔，并将该异类分型作为起点、新分型作为终点。

若新分型与缓存中所有分型类型相同（不能成笔），则继续缓存，等待后续反向分型。

---

## 5. 成笔判断

判断两个异类分型是否能连接成笔，需要依次通过以下检查：

### 5.1 基础条件

起点和终点必须是不同类型分型（一顶一底）。

### 5.2 跨度检查

在非严格模式下，要求：

- 合并K线跨度至少为 3（`终点索引 - 起点索引 ≥ 3`）。
- 中间包含的原始K线数量至少为 3。

在严格模式下，只要求合并K线跨度至少为 4。

若 `缺口计为K线` 配置开启，则中间存在缺口的相邻合并K线在计跨度时额外加 1。加速优化：跨度已 ≥ 4 时可跳过精确缺口检查。

### 5.3 价格区间检查

上升笔：终点的最高价必须高于起点的最低价（即顶不能在底内部或更低）。

下降笔：终点的最低价必须低于起点的最高价（即底不能在顶内部或更高）。

### 5.4 分型有效性检查

见第 6 节。四种检查方法可选：`宽松`、`半`、`严格`、`完全`。

### 5.5 终点峰值检查

若 `笔终点为峰值` 配置开启，则笔终点必须是起点到终点范围内的最高点或最低点：

- 上升笔的终点必须是区间内的最高价。
- 下降笔的终点必须是区间内的最低价。

---

## 6. 分型有效性检查

分型有效性检查由起点分型（`自身`）调用，检查候选终点（`第二元素`）是否与起点构成有效的顶底关系。四种方法的差异在于检查的范围。

### 6.1 顶分型起点

起点为顶分型，候选为底分型。以下条件满足时通过检查。

**虚笔处理**：若用于虚笔，候选终点的方向必须为向下（替代分型判断）。

**方法 `宽松`**

只看两个分型自身：终点低点 > 起点高点。

判断：`自身.低 > 第二元素.高 且 第二元素.低 < 自身.低`

**方法 `半`（默认）**

看终点前一根和起点后一根：

```
第二元素高点 = 最大值(第二元素.前一根.高, 第二元素.高)
自身低点 = 最小值(自身.低, 自身.下一根.低)
```

判断：`自身.高 > 第二元素高点 且 第二元素.低 < 自身低点`

**方法 `严格`**

看两个分型的三元素（分型自身及前后各一根）：

```
第二元素高点 = 最大值(第二元素.前一根.高, 第二元素.高, 第二元素.下一根.高)
自身低点 = 最小值(自身.前一根.低, 自身.低, 自身.下一根.低)
```

判断：`自身.高 > 第二元素高点 且 第二元素.低 < 自身低点`

**方法 `完全`**

要求两个分型区间完全分离：

判断：`自身.低 > 第二元素.高`

### 6.2 底分型起点

起点为底分型，候选为顶分型。规则与顶分型对称，将高/低、大/小、顶/底分别取反即可。

**虚笔处理**：若用于虚笔，候选终点的方向必须为向上。

**方法 `宽松`**：`自身.低 < 第二元素.低 且 第二元素.高 > 自身.高`

**方法 `半`**：

```
第二元素低点 = 最小值(第二元素.前一根.低, 第二元素.低)
当前高点 = 最大值(自身.高, 自身.下一根.高)
```

判断：`自身.低 < 第二元素低点 且 第二元素.高 > 当前高点`

**方法 `严格`**：

```
第二元素低点 = 最小值(第二元素.前一根.低, 第二元素.低, 第二元素.下一根.低)
当前高点 = 最大值(自身.前一根.高, 自身.高, 自身.下一根.高)
```

判断：`自身.低 < 第二元素低点 且 第二元素.高 > 当前高点`

**方法 `完全`**：`自身.高 < 第二元素.低`

### 6.3 方法对比

| 方法 | 检查范围 | 特点 |
| --- | --- | --- |
| `宽松` | 只看两个分型自身 | 较宽松 |
| `半` | 看候选终点前一根和起点后一根 | 默认折中 |
| `严格` | 看两个分型三元素 | 更严格 |
| `完全` | 两个分型区间完全分离 | 最保守，过滤最多 |

---

## 7. 终点峰值检查

仅当 `笔终点为峰值 = 真` 时启用。

检查从笔起点到终点范围内，终点是否为最高点或最低点。

若起点为底分型（上升笔），则从起点的下一根合并K线开始，逐根向后检查是否出现高于候选终点价格的高点。若中间没有任何高点高于候选终点，则通过峰值检查。

若起点为顶分型（下降笔），则检查中间是否出现低于候选终点价格的低点。若没有任何低点低于候选终点，则通过峰值检查。

---

## 8. 同类分型更新终点

当新分型与 `最后终点` 类型相同（同为顶或同为底）时，不创建新笔，而是尝试更新最后一笔的终点。

更新条件：

- 若最后一笔方向为向上，新分型必须为顶分型且其高点不低于最后一笔的当前终点高点。
- 若最后一笔方向为向下，新分型必须为底分型且其低点不高于最后一笔的当前终点低点。

满足条件时，将最后终点更新为新分型，并将最后一笔的终点设置为该新分型。

---

## 9. 次高/次低修正

当 `允许次峰值 = 假` 时，在某些情况下需要撤销上一笔并创建一个更合理的笔。

在以下所有条件满足时触发修正：

1. 笔列表中至少有两条笔。
2. 待处理的高点或低点与上一笔的方向匹配（向上笔不处理低于起点的低点，向下笔不处理高于起点的高点）。
3. 待处理的高点或低点与前一笔的起点相比是有效峰值。
4. 上一笔的终点值没有跨过前一笔的起点值（即笔的方向性成立）。

修正执行时，先移除最后一笔，然后尝试用待处理的分型更新新的最后一笔的终点。若更新成功则修正完成，否则恢复被移除的笔。

---

## 10. 虚笔机制

虚笔用于实时行情，表示一条尚未确认的笔，可能随最新K线变化。

### 10.1 删除或恢复虚笔

每次确认笔计算前，先检查笔列表中的最后一条笔是否为虚笔。若是虚笔：

- 若该虚笔曾记录过若干确认终点（即虚笔是把一条确认笔的终点替换后形成的），则恢复到最后一条确认终点，并将多余的确认终点依次创建为确认笔。
- 若虚笔没有记录过确认终点（即直接新增的未确认笔），则直接删除。

### 10.2 添加或更新虚笔

输入最新的合并K线后：

**情况一：沿最后一笔方向延伸**

若最新K线沿最后一笔的方向继续创新高（向上笔）或创新低（向下笔），则更新最后一笔的虚拟终点，不创建新笔。

**情况二：尝试创建新虚笔**

从最新K线向前追溯，检查能否与最后一笔的确认终点形成一条新虚笔。若能成笔（通过 `可否成笔` 检查，标记为虚笔），则创建一条新虚笔。若在追溯过程中满足修正条件，也可触发修正。

### 10.3 虚笔状态更新

更新虚笔终点时，将被替换前的终点记录下来，以便后续恢复。当虚笔最终被确认时，恢复被替换的终点链。

# 笔识别模式说明

## 三种页面模式与配置映射

页面模式是对配置参数的简化映射。

| 页面模式 | 建议配置 | 用途 |
| --- | --- | --- |
| 宽松 | `笔算法=顶底即成笔` 或 `正常 + 是否严格=假 + 笔终点为峰值=假` | 观察短线细节 |
| 标准 | `正常 + 是否严格=假 + 笔分型检查=半 + 笔终点为峰值=真` | 实时默认、常规结构 |
| 严格 | `正常 + 是否严格=真 + 笔分型检查=半/严格 + 笔终点为峰值=真` | 保守复盘、结构确认 |

注意：

- 默认是 `正常 + 是否严格=真 + 笔分型检查=半 + 缺口计为K线=真 + 笔终点为峰值=真 + 允许次峰值=真`。
- 默认配置虽然 `是否严格=真`，但 `笔分型检查=半`，不是最极端的 `严格` 分型检查。
- `完全` 是特殊强过滤口径，不建议作为默认模式。

---

## 实现顺序建议

若从零实现，建议按以下顺序：

1. 实现包含处理后的 `合并K线` 链表结构，补齐 `前一根/下一根/索引/高/低/分型/方向/原始列表`。
2. 实现分型识别，给每个 `合并K线` 标记 `顶/底/未知`。
3. 实现 `笔对象`，包括方向、起终点、确认状态和虚笔状态。
4. 实现 `笔列表类` 状态：`笔列表`、`最后终点`、`候选分型列表`。
5. 实现跨度检查和缺口跨度计算。
6. 实现分型有效性检查的 `宽松/半/严格/完全` 四种口径。
7. 实现终点峰值检查。
8. 实现成笔总判断。
9. 实现第一笔生成。
10. 实现同类分型更新终点。
11. 实现次高/次低修正。
12. 实现确认笔更新总流程。
13. 实现虚笔删除、恢复和添加。
14. 最后实现总入口更新流程。

---

## 当前实现原则

- 确认笔参与线段、中枢、买卖点计算。
- 虚笔用于实时显示和滚动更新。
- 是否让虚笔参与后续结构，应由上层配置控制，默认不参与。
- 应采用默认配置而不是页面简化配置，以获得更严格的结构结果。
