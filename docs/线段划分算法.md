# 线段划分算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 输入

- `笔列表[]`：笔的序列。
- 已知前一段线段的方向。

### 输出

- `线段列表[]`：线段列表，每个线段包含若干笔。

### 线段破坏判定（两种情况）

> **术语区分**：以下"情况一/二"基于第78课「笔破坏与线段破坏的异同」的分类（是否直接笔破坏）。第67课的"第一种/第二种情况"则是特征序列分型中有无缺口的分类，两者是不同的分类维度，不可混淆。

#### 情况一：新笔直接笔破坏前段

当一段向上的线段遇到第一根方向向下的笔，且该向下笔的低点跌破了前一段倒数第二笔（向上笔）的低点时，视为直接笔破坏。之后需要确认特征序列中出现了顶分型，才能确认线段被破坏。

反之，向下线段被向上笔直接破坏的判断同理。

#### 情况二：新笔未直接笔破坏前段，后续确认

若新方向的第一笔没有直接破坏前一段，则继续观察后续走势。当特征序列出现分型（顶分型或底分型），且特征序列第一元素与第二元素之间 **没有缺口**（即属于同一特征序列）时，确认线段被破坏。

### 完整划分流程

从第一笔开始，逐步向后扫描。

将当前笔加入当前线段的笔序列。每加入一笔后，检查当前线段是否已被破坏。

若检测到破坏（类型一或类型二），则将当前线段中除破坏笔及其后一笔之外的笔确认为一个已完成线段。破坏笔作为新线段的第一笔，方向取反，继续向后扫描。

所有笔处理完毕后，将最后一段不足 3 笔的线段也纳入结果。

### 线段破坏条件总结

| 条件 | 类型一 | 类型二 |
| --- | --- | --- |
| 第一笔是否直接笔破坏前段 | 是 | 否 |
| 特征序列一二元素间是否有缺口 | 无关 | 无缺口 |
| 确认方式 | 特征序列出现分型 | 特征序列出现分型 |
