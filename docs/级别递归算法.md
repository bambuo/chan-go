# 级别递归算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 核心思想

高级别走势由低级别走势 **递归构成**，而非人为设定时间周期。

### 算法步骤

从最基础的原始K线开始，逐层向上构建：

**第 0 层 — K线**

原始K线数据作为基础输入。

**第 1 层 — 分型和笔**

对原始K线做包含处理，得到合并K线序列。在合并K线上识别顶底分型，再由分型序列连接成笔。

**第 2 层 — 线段**

根据笔序列划分线段，每个线段包含方向一致的若干笔。

**第 3 层 — 最小级别中枢**

在线段序列上识别中枢：连续三段的重叠区域为一个中枢。

**第 4 层以上 — 递归上升**

将当前层的中枢视为"笔"，用它们构造更高一级的线段，再从中识别更高一级的中枢。重复此过程，直到无法形成至少三个中枢为止。

### 级别与时间周期（经验对应，非定义）

| 递归级别 | 常见时间周期 | 常用均线参数 |
| --- | --- | --- |
| 最小级（笔） | 1分钟图 | 无 |
| 低一级（线段） | 5分钟图 | 5、13 |
| 中级别（中枢） | 30分钟图 | 34、55 |
| 高级别 | 日线 | 89、144 |
| 更高级别 | 周线 | 233 |

注意：这些时间周期只是经验对应，并非级别的定义。级别是递归出来的，不是时间决定的。
