# 分型识别算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 输入

- `合并K线[]`：经过包含处理后的合并K线序列。

### 输出

- `顶分型列表[]`：顶分型列表，每个元素为 `(索引, 高, 低)`。
- `底分型列表[]`：底分型列表。

### 算法步骤

合并K线数量不足 3 根时直接返回空列表。

对每相邻三根合并K线（索引 i-1, i, i+1）：

**顶分型判定**

若中间K线的 **高点** 同时高于左侧和右侧K线的高点，
且中间K线的 **低点** 同时高于左侧和右侧K线的低点，
则该中间K线为一个顶分型，记录其索引、高点、低点。

条件表达式为：

```
中间高 > 左侧高 且 中间高 > 右侧高
中间低 > 左侧低 且 中间低 > 右侧低
```

两者缺一不可（原文定义：顶分型要求中间K线高点最高且低点也最高）。

**底分型判定**

若中间K线的 **低点** 同时低于左侧和右侧K线的低点，
且中间K线的 **高点** 同时低于左侧和右侧K线的高点，
则该中间K线为一个底分型。

条件表达式为：

```
中间低 < 左侧低 且 中间低 < 右侧低
中间高 < 左侧高 且 中间高 < 右侧高
```

**互斥性**

顶分型和底分型使用 `否则若` 判定，确保同一根合并K线不会同时既是顶分型又是底分型。

### 分型有效性过滤（实践补充，非原文明确定义）

实践中常用的确认方式：

- 顶分型形成后，需后续K线跌破顶分型第三根K线的低点来确认。
- 底分型形成后，需后续K线升破底分型第三根K线的高点来确认。
- 否则该分型可能为"未确认"状态，可暂不生效。
