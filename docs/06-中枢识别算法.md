# 中枢识别算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 定义

**中枢** = 连续至少 3 笔（或 3 段次级别走势）的重叠区间。

### 算法步骤

扫描线段（或笔）序列，每次取连续的三段。

**计算重叠区间**

对三段各自的最高价取最小值，得到中枢上沿 ZG（高点中的最低者）；对三段各自的最低价取最大值，得到中枢下沿 ZD（低点中的最高者）。若 ZG > ZD，则三段存在有效重叠。

若不存在有效重叠，跳过这组，从下一段开始继续扫描。

**中枢延伸**

若存在有效重叠，形成一个初始中枢（包含这三段）。从后续第四段开始，依次检查每段是否与当前中枢区间有交集（即该段的最高价 ≥ ZD 且该段的最低价 ≤ ZG）。若有交集，则将该段纳入中枢延伸，继续检查下一段。

**中枢完成判定（第三类买卖点确认）**

当出现一段完全脱离中枢区间（该段最高价 < ZD 或最低价 > ZG）时，延伸暂时终止，但不立即标记中枢完成。需再检查下一段（回抽/回升段）：

- 若回抽段重新回到中枢区间（与 ZG/ZD 有交集），则中枢继续延伸（吸收脱离段及回抽段）。
- 若回抽段也未回到中枢区间，则中枢被 **第三类买卖点确认完成**（`Completed = true`）。此时脱离段 + 回抽段均不纳入中枢。

跳过已纳入中枢延伸的段，从中枢结束后的下一段开始扫描下一个中枢。

### 中枢升级

- **中枢延伸**：围绕中枢的震荡持续延伸，社区经验值为 9 段延伸后升级为高级别中枢，原文未明确指定具体段数。
- **中枢扩展**：两个同级别中枢的完整波动区间有重叠时，升级为更高级别中枢。
