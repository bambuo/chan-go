# 走势类型分类算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 定义

- **趋势**：含两个及以上 **互不重叠** 的同向中枢。
- **盘整**：只含一个中枢。

### 算法步骤

**边界情况**

- 若无中枢，走势类型不明朗。
- 若只有一个中枢，为盘整。

**多中枢判定**

若有两个或以上中枢，需要同时检查两个条件：

**同向性**：相邻中枢的方向必须相同。遍历所有相邻中枢对，若任一方向不同，则不构成趋势。

**非重叠性**：相邻中枢的波动区间不能重叠。若任一相邻中枢的区间有重叠，则视为中枢扩展，整体仍按盘整处理。

同时满足同向且非重叠时，按第一个中枢的方向判定为上涨趋势或下跌趋势。否则为盘整。
