# 笔识别算法规范

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

---

## 1. 基本对象

### 1.1 合并K线

笔识别基于包含处理后的合并K线，记为 `合并K线`。

每个 `合并K线` 至少包含：

| 字段 | 含义 |
| --- | --- |
| `索引` | 合并K线索引，从 0 递增 |
| `高` | 合并K线最高价 |
| `低` | 合并K线最低价 |
| `分型` | 分型类型：`顶`、`底`、`未知` |
| `方向` | 合并K线方向：`上`、`下` 等 |
| `前一根` | 前一根合并K线 |
| `下一根` | 后一根合并K线 |
| `原始列表` | 该合并K线包含的原始K线列表 |

辅助计算：

- 合并K线最高价：取其 `原始列表` 中各原始K线的最高价的最大值。
- 合并K线最低价：取其 `原始列表` 中各原始K线的最低价的最小值。
- 缺口判定：若两根相邻合并K线的真实价格区间（各自最高价到最低价）完全不重叠，则判定为存在缺口。

### 1.2 分型类型

| 类型 | 含义 |
| --- | --- |
| `顶` | 顶分型 |
| `底` | 底分型 |
| `未知` | 非分型 |

### 1.3 笔对象

每条笔至少包含：

| 字段 | 含义 |
| --- | --- |
| `起点合并K线` | 笔起点合并K线 |
| `终点合并K线` | 笔终点合并K线 |
| `索引` | 笔序号 |
| `方向` | 笔方向：`上` 或 `下` |
| `是否确认` | 是否为确认笔 |
| `确认终点[]` | 虚笔曾经替换过的确认终点 |
| `前一笔` | 前一笔 |
| `后一笔` | 后一笔 |

方向规则：

笔的方向由起点分型决定：起点为底分型时笔方向为向上，起点为顶分型时笔方向为向下。

价格规则：

- 向上笔的起点价格取起点合并K线的低点，终点价格取终点合并K线的高点。
- 向下笔的起点价格取起点合并K线的高点，终点价格取终点合并K线的低点。

合法性检查：向上笔必须确保起点低点 < 终点高点；向下笔必须确保起点高点 > 终点低点。

## 2. 配置项

### 2.1 配置表

| 配置 | 默认值 | 含义 |
| --- | --- | --- |
| `是否严格` | `真` | 是否使用严格跨度规则（严格=4，非严格=3） |

### 2.2 跨度说明

本项目仅实现严笔（始终检查跨度和价格区间），不实现顶底即成笔的宽笔模式。

- **严格模式**（默认）：合并K线跨度 ≥ 4。
- **非严格模式**：合并K线跨度 ≥ 3，且中间原始K线 ≥ 3。

---

## 3. 主状态机

笔识别是一个逐步推进的状态机，维护以下核心状态：

- `笔列表[]`：已生成的笔（可能包含最后一条虚笔）。
- `最后终点`：最后一笔的尾部分型（合并K线）。
- `候选分型列表[]`：第一笔生成前暂存的候选分型。

### 3.1 总入口

输入：一根已可确认分型的合并K线（倒数第二根），以及最新合并K线（用于生成或更新虚笔）。可选择是否计算虚笔。

逻辑：先执行确认笔更新流程。若需要计算虚笔，再执行虚笔更新流程。返回是否有任何变化。

### 3.2 确认笔更新流程

每次计算确认笔前，先清理上一轮产生的虚笔。

对输入的合并K线：

- **若无分型**：检查最后一笔的终点K线索引是否有变化即可。
- **若笔列表为空或最后终点为空**：尝试用该合并K线创建第一笔。
- **若该合并K线与最后终点类型相同**（同为顶或同为底）：尝试更新最后一笔的终点。
- **若该合并K线可与最后终点成笔**：创建一条新确认笔，将该合并K线设为新的最后终点。
- **若可触发更新峰值修正**：执行修正逻辑。

---

## 4. 第一笔生成

第一笔不是直接从最早两个不同分型生成，而是先缓存若干个候选分型，以便找到更合适的起点。

当笔列表为空时，每个到来的合并K线如果是有分型的，先进入 `候选分型列表`。每当新分型到来，检查它是否与 `候选分型列表` 中的异类分型成笔。若能成笔，则创建第一笔，并将该异类分型作为起点、新分型作为终点。

若新分型与缓存中所有分型类型相同（不能成笔），则继续缓存，等待后续反向分型。

---

## 5. 成笔判断

判断两个异类分型是否能连接成笔，需要依次通过以下检查：

### 5.1 基础条件

起点和终点必须是不同类型分型（一顶一底）。

### 5.2 跨度检查

在非严格模式下，要求：

- 合并K线跨度至少为 3（`终点索引 - 起点索引 ≥ 3`）。
- 中间包含的原始K线数量至少为 3。

在严格模式下，只要求合并K线跨度至少为 4。

### 5.3 价格区间检查

上升笔：终点的最高价必须高于起点的最低价（即顶不能在底内部或更低）。

下降笔：终点的最低价必须低于起点的最高价（即底不能在顶内部或更高）。

### 5.4 分型有效性

缠论原文未定义分型有效性检查。分型由 AND 双条件保证有效性后即可成笔，无需额外验证（实现中该检查始终通过）。

### 5.5 价格合法性

- 上升笔：终点的最高价必须高于起点的最低价（`end.High > start.Low`）。
- 下降笔：终点的最低价必须低于起点的最高价（`end.Low < start.High`）。

---

## 6. 同类分型终点更新

当新分型与 `最后终点` 类型相同（同为顶或同为底）时，不创建新笔，而是尝试更新最后一笔的终点。

更新条件：

- 若最后一笔方向为向上，新分型必须为顶分型且其高点不低于最后一笔的当前终点高点。
- 若最后一笔方向为向下，新分型必须为底分型且其低点不高于最后一笔的当前终点低点。

满足条件时，将最后终点更新为新分型，并将最后一笔的终点设置为该新分型。

## 7. 虚笔机制

虚笔用于实时行情，表示一条尚未确认的笔，可能随最新K线变化。

### 10.1 删除或恢复虚笔

每次确认笔计算前，先检查笔列表中的最后一条笔是否为虚笔。若是虚笔：

- 若该虚笔曾记录过若干确认终点（即虚笔是把一条确认笔的终点替换后形成的），则恢复到最后一条确认终点，并将多余的确认终点依次创建为确认笔。
- 若虚笔没有记录过确认终点（即直接新增的未确认笔），则直接删除。

### 10.2 添加或更新虚笔

输入最新的合并K线后：

**情况一：沿最后一笔方向延伸**

若最新K线沿最后一笔的方向继续创新高（向上笔）或创新低（向下笔），则更新最后一笔的虚拟终点，不创建新笔。

**情况二：尝试创建新虚笔**

从最新K线向前追溯，检查能否与最后一笔的确认终点形成一条新虚笔。若能成笔（通过 `可否成笔` 检查，标记为虚笔），则创建一条新虚笔。若在追溯过程中满足修正条件，也可触发修正。

### 10.3 虚笔状态更新

更新虚笔终点时，将被替换前的终点记录下来，以便后续恢复。当虚笔最终被确认时，恢复被替换的终点链。

# 笔识别模式说明

## 两种跨度模式

| 模式 | 配置 | 用途 |
| --- | --- | --- |
| 标准（非严格） | `是否严格=假` | 实时默认、常规结构 |
| 严格（默认） | `是否严格=真` | 保守复盘、结构确认 |

注意：

- 默认 `是否严格=真`。
- `是否严格` 控制的是跨度要求（严格=4，非严格=3），原文仅规定"至少有一K线"。

---

## 实现顺序建议

若从零实现，建议按以下顺序：

1. 实现包含处理后的 `合并K线` 链表结构，补齐 `前一根/下一根/索引/高/低/分型/方向/原始列表`。
2. 实现分型识别，给每个 `合并K线` 标记 `顶/底/未知`。
3. 实现 `笔对象`，包括方向、起终点、确认状态和虚笔状态。
4. 实现 `笔列表类` 状态：`笔列表`、`最后终点`、`候选分型列表`。
5. 实现跨度检查和缺口跨度计算。
6. 实现成笔总判断（含跨度检查、价格区间检查）。
7. 实现第一笔生成。
8. 实现同类分型更新终点。
9. 实现确认笔更新总流程。
10. 实现虚笔删除、恢复和添加。
11. 最后实现总入口更新流程。

---

## 当前实现原则

- 确认笔参与线段、中枢、买卖点计算。
- 虚笔用于实时显示和滚动更新。
- 是否让虚笔参与后续结构，应由上层配置控制，默认不参与。
- 应采用默认配置而不是页面简化配置，以获得更严格的结构结果。
