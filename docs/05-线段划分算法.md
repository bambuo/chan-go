# 线段划分算法

> 本文遵循 [中文伪代码规范](./中文伪代码规范.md)。

### 输入

- `笔列表[]`：笔的序列。
- 已知前一段线段的方向。

### 输出

- `线段列表[]`：线段列表，每个线段包含若干笔。

### 线段破坏判定（两种情况）

> **术语区分**：以下"情况一/二"基于第67课的特征序列分型分类（按分型第1、2元素间是否有缺口）。第78课另有"
> 笔破坏与线段破坏的异同"的讨论（是否直接笔破坏），是不同的分析视角，不可混淆。判定线段是否破坏的唯一标准是第67课的两种情况。

> **核心前提**（第67课）：**出现特征序列的分型，是线段结束的前提条件。**

#### 特征序列的方向

- 以向上笔开始的线段，其特征序列 = 该线段中所有**向下笔**，**只考察顶分型**。
- 以向下笔开始的线段，其特征序列 = 该线段中所有**向上笔**，**只考察底分型**。

特征序列须先做包含处理得到标准特征序列，再在标准特征序列上判定分型（分型定义见 [04-特征序列算法.md](./04-特征序列算法.md)
，双AND定义，与普通K线分型一致）。

#### 情况一：分型第1、2元素间无缺口

当标准特征序列出现分型，且该分型的**第1元素与第2元素之间没有缺口**
（两者价格区间有重合）时，线段在该分型的极值点（顶分型的高点 / 底分型的低点）处结束，该极值点是线段的终点。

#### 情况二：分型第1、2元素间有缺口

当标准特征序列出现分型，但该分型的**第1元素与第2元素之间存在缺口**（两者价格区间完全不重叠）时，不能立即确认线段结束。此时需从该分型极值点开始的反向笔序列（
**第二特征序列**）再出现分型，才能确认线段在该极值点处结束。

> 第67课强调：第二种情况下，第二个序列中的分型**不分第一二种情况，只要有分型就可以**。第二特征序列不要求封闭第一特征序列相应的缺口。

> 第78课补充：第二种情况的第二特征序列，其包含处理方向与原线段方向**一致**（与第一特征序列相反），且必须严格按包含关系处理。

### 完整划分流程

从第一笔开始，逐步向后扫描。

对每个可能的转折点（出现反向笔时），假设其为两线段分界点，按上述两种情况考察：

1. 构建当前线段的第一特征序列，检测分型。
2. 若无分型 → 当前线段继续延伸，等待更多笔。
3. 若有分型，判分型第1、2元素间是否有缺口：
    - 无缺口（情况一）→ 确认线段在分型极值点结束。
    - 有缺口（情况二）→ 记录待确认态，等待第二特征序列出现分型后再确认。

确认破坏后，分型极值点之前（含）的笔归属原线段（原线段完成），之后的笔归属新线段（方向取反），继续向后扫描。

### 线段破坏条件总结

| 条件             | 情况一   | 情况二       |
|----------------|-------|-----------|
| 特征序列是否出现分型     | 是（前提） | 是（前提）     |
| 分型第1、2元素间是否有缺口 | 无缺口   | 有缺口       |
| 是否需要第二特征序列     | 否     | 是（需出现分型）  |
| 第二特征序列分型是否判缺口  | 不适用   | 不判（有分型即可） |

### 线段方向约束（第78课）

- 以向上笔开始的线段必结束于向上笔（终点是顶）；以向下笔开始的线段必结束于向下笔（终点是底）。
- 同一线段两端的顶必高于底。
- 前后相邻两线段方向相反，且后线段必须破坏前线段（破坏关系可逆时间传递）。
